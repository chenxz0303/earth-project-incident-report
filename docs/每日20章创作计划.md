# 每日20章自动创作计划

_从第002章开始，在 Codex 本地自动任务运行时每天顺序完成最多20章；质量门禁优先于数量。_

---

## 📋 总体安排

| 创作日 | 计划章节 | 数量 | 主要阶段 |
|---|---|---:|---|
| 第1日 | 002—021 | 20章 | 第一卷现实异常升级 |
| 第2日 | 022—041 | 20章 | 第一卷收束、第二卷舰上调查 |
| 第3日 | 042—061 | 20章 | 第二卷责任线、第三卷地球修复 |
| 第4日 | 062—081 | 20章 | 第三卷危机收束、第四卷监管预审 |
| 第5日 | 082—101 | 20章 | 第四卷证据资格、第五卷任务组启动 |
| 第6日 | 102—121 | 20章 | 第五卷责任复现与证据闭环 |
| 第7日 | 122—141 | 20章 | 第五卷收束、第六卷修复与听证 |
| 第8日 | 142—150 | 9章 | 全书结局、总连续性与发布终审 |

计划按“当前最新正式章节”继续，不按日历补欠账。某天未运行时，下一次仍只处理最多20章，不把两天任务合并成40章。

## 🔄 每日批次

每天拆成4批，每批最多5章：

| 批次 | 当日范围 | 完成动作 |
|---|---|---|
| A | 第1—5章 | 逐章写作、逐章维护档案、逐章测试；完成后提交并同步 |
| B | 第6—10章 | 同上 |
| C | 第11—15章 | 同上 |
| D | 第16—20章 | 同上；结束后执行当日总审查 |

第8日只执行A、B两批，完成第150章后进行全书终审。

## ✍️ 单章硬要求

- 依据 `docs/章节规划.md` 和对应详细场景卡创作
- 每章目标约3000—4000字，优先遵守该章任务书中的具体范围
- 使用第三人称限知，主要跟随陈默
- 数字遵守阿拉伯数字规则
- 不提前泄露后续真相，不把规划内容当成已发生剧情
- 每章更新章节摘要、时间线、人物状态、伏笔表和连续性台账
- 关键设定实际变化时更新世界观规则
- 更新章节 JSON、目录、书籍信息和上一章/下一章关系
- 每章通过自动检查后才能进入下一章

## ✅ 每批发布规则

1. 每完成1章，先在本地运行完整检查
2. 每完成5章，创建1次批次提交
3. 同步 GitHub `main` 与 Gitee `main`
4. 等待 GitHub Pages 自动部署成功
5. 验证 Gitee Raw 目录和批次最后一章可以读取
6. 确认两个仓库提交编号一致，再开始下一批

## ⛔ 自动停止条件

出现以下任一情况，当天立即停止，不为凑够20章继续生成：

- 工作区存在无法确认来源的用户改动
- 当前章节与时间线、人物已知信息或权限账本冲突
- 正文质量检查或项目自动检查失败
- 章节字数、标题、目录或维护档案不一致
- GitHub 或 Gitee 推送失败
- GitHub Pages 部署失败
- 远程章节或书源地址无法读取
- 正式章节已经达到第150章

停止后必须报告已完成章节、失败原因、最后成功提交和下次应从哪一章继续，不得静默跳过问题。

## 🕘 执行时间

- 每天09:00尝试启动
- 使用本机 Codex 项目环境执行
- Codex 或本机环境不可用时不强行补跑
- 每次最多完成20章
- 完成第150章后只报告全书已完成，不再生成新章节
