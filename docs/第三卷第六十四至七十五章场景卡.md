# 第三卷第六十四至七十五章场景卡

> 本文件是第三卷后半段场景设计，不是小说正文。第 64—75 章负责完成台风—桥梁—热网耦合危机，证明地球居民具备样本内自救能力，随后揭示非法文明决策数据交易和事故前就已存在的地球终止预案。

## 第 064 章：风、桥和一锅热水

### 章节任务

- 完成第三卷多物理场中段高潮。
- 让四名工程角色真正协作，而不是陈默一人解题。
- 以拆开错误耦合代替全局一键修复。

### 场景拆分

| 场景 | 主要动作 | 结果 |
|---|---|---|
| 1. 联合模型 | 唐可给阵风输入，坎提斯给桥梁响应，乌尔澄给热网流量。 | 确认三系统被节点共享参数锁在一起。 |
| 2. 分工方案 | 陈默定义权限范围，地球单位负责限流、分区和现场支撑。 | 每个人只控制自己懂的部分。 |
| 3. 先卸热 | 热网分区泄压，让节点失去异常热源。 | 桥梁阻尼短暂继续下降。 |
| 4. 再拆耦合 | 陈默取消节点的共享压缩参数，让风载、耗散和热流恢复独立。 | 桥梁响应回到可由临时阻尼控制的范围。 |
| 5. 台风通过 | 常规应急措施承受剩余风雨，城市没有被参数修改成“无风区”。 | 危机解除，权限预算接近耗尽。 |

### 笑点与专业

- 笑点：三专业终于同意用一张图，但仍争论颜色代表谁；标题像做饭却没人有空吃饭。
- 专业点：流固热耦合与错误降阶；解决方法是拆开不该存在的连接，而非把所有数值强行调正常。
- 章尾：危机中大量普通维护人员的行动被系统记录为“样本内自主纠错”。

## 第 065 章：没有权限的维护队

### 章节任务

- 证明地球居民不需要现实修改权限也能参与修复。
- 埋设地球自治评级伏笔。
- 给大型危机后的普通人视角留出空间。

### 场景拆分

| 场景 | 主要动作 | 结果 |
|---|---|---|
| 1. 行动记录 | 团队整理桥梁、热网、气象和医院现场处置。 | 多项关键措施来自不知真相的普通人员。 |
| 2. 系统误判 | 外部系统最初把这些行动归为随机样本噪声。 | 陈默要求按实际因果贡献重新分类。 |
| 3. 可重复协作 | 唐可证明不同单位在有限信息下仍能形成有效闭环。 | 行动并非偶然碰巧。 |
| 4. 自治评级 | 地球项目的自治能力指标首次上升。 | F-011 正式埋设。 |
| 5. 陈默的边界 | 有人建议今后全部交给地球人员，陈默指出他们需要真实信息和安全资源。 | 自治不等于把外部责任甩回地球。 |

### 笑点与专业

- 笑点：宇宙系统把维修班组的微信群协作识别成“非标准分布式控制协议”；赵启明认为这至少证明群消息不是全无价值。
- 专业点：韧性、冗余和分布式响应；真正稳健的系统不是只有一个超级控制器。
- 章尾：危机结束后，芙娅收到调离事故组的内部预警，却没有告诉陈默。

## 第 066 章：联络官的夜宵审批

### 章节任务

- 在高压主线后提供人物缓冲。
- 推进陈默与芙娅的私人信任，但保持克制。
- 埋设芙娅未来职责冲突。

### 场景拆分

| 场景 | 主要动作 | 结果 |
|---|---|---|
| 1. 强制休息 | 赵启明要求陈默和唐可至少暂停工作一小时。 | 陈默第一次离开监测屏幕。 |
| 2. 芙娅尝试夜宵 | 受限投影不能正常进食，只能通过感官映射体验少量味觉。 | 地球习惯成为轻松交流入口。 |
| 3. 工作外问题 | 芙娅询问陈默为何总先怀疑自己做错。 | 陈默只给部分真实回答，不进行长篇自白。 |
| 4. 调离预警 | 芙娅收到信息后迅速切回工作状态。 | 她暂时隐瞒可能被调离。 |
| 5. 新证据通知 | 节点数据包完成解码，出现与目标覆盖相同的编码水印。 | 休息被迫结束。 |

### 笑点与感情控制

- 夜宵报销系统不承认“投影体验”属于餐费；芙娅把辣度当成设备报警等级。
- 暧昧只体现在两人开始关心对方工作之外的状态，不写告白、脸红或强行肢体接触。
- 章尾转回主线，避免感情段落独立成番外感。

## 第 067 章：相同的水印

### 章节任务

- 用数字指纹连接地球数据节点与发射目标覆盖。
- 把“同一系统”与“同一操作者”区分开。
- 推进归零服务联合体的间接存在。

### 场景拆分

| 场景 | 主要动作 | 结果 |
|---|---|---|
| 1. 数据包解码 | 恢复节点上传包头和压缩目录。 | 每个数据包都有固定编码水印。 |
| 2. 覆盖指令比对 | 芙娅调出发射前 0.8 秒残留指令。 | 水印格式完全一致。 |
| 3. 排除通用协议 | 调查归零服务协议版本与其他客户样本。 | 水印属于特定处置工具链，不是全行业通用。 |
| 4. 操作者边界 | 陈默提醒同一工具不能直接证明同一人。 | 结论限定为同一技术系统或供应链。 |
| 5. 目录入口 | 数据包目录含“决策压力”“合作崩溃”“技术跃迁”等分类。 | 采集内容超过普通备份。 |

### 笑点与专业

- 笑点：工具水印本来用于证明知识产权，现在先证明工具参与了事故；供应商把它称为品牌一致性。
- 专业点：数字指纹、工具链归属、证据边界。
- 章尾：一组社会层数据表现为排队顺序长期偏向同一类人。

## 第 068 章：排队总是同一个人先到

### 章节任务

- 引入社会概率层异常，扩大数据抽取范围。
- 保持自由意志议题开放，不把人写成参数木偶。
- 证明节点在主动制造可比较的决策情景。

### 场景拆分

| 场景 | 主要动作 | 结果 |
|---|---|---|
| 1. 小概率偏置 | 城市多个无关排队系统出现相似优先顺序。 | 单次看似巧合，长期统计显著。 |
| 2. 排除算法 | 唐可和陈默排查支付、预约和身份规则。 | 偏置发生在系统决策之前。 |
| 3. 节点实验组 | 数据目录将这些区域标为“轻压力合作基线”。 | 节点不仅观察，还在设置条件。 |
| 4. 不直接改人 | 外部系统建议恢复个体选择概率，陈默拒绝修改人格或具体决定。 | 只移除外部偏置场。 |
| 5. 行为仍不同 | 偏置移除后，人们并没有变成统一随机结果。 | 个体选择不等于可完全预测参数。 |

### 笑点与专业

- 笑点：最先发现偏置的是总抢不到食堂限量套餐的同事；系统把插队投诉归为文明协作数据。
- 专业点：统计偏差、控制变量、概率不等于命运。
- 章尾：陈默发现节点保存了数十年的地球危机决策实验目录。

## 第 069 章：谁拥有我们的选择

### 章节任务

- 让陈默与芙娅围绕数据权利发生原则分歧。
- 推进芙娅对项目法的反思。
- 为数据交易揭示增加伦理重量。

### 场景拆分

| 场景 | 主要动作 | 结果 |
|---|---|---|
| 1. 项目所有权 | 芙娅按现行规则说明项目所有者通常拥有运行数据。 | 陈默认为居民承担后果，不能被视为无权利传感器。 |
| 2. 合法不等于授权 | 陈默追问地球居民是否同意危机被制造并出售。 | 芙娅无法给出有效授权记录。 |
| 3. 职责争执 | 芙娅担心在调查完成前使用“非法”会污染结论。 | 陈默认为过度中性也可能保护施害者。 |
| 4. 共同措辞 | 两人暂定“未经样本内知情的外部数据提取”。 | 保留事实判断，暂缓法律定性。 |
| 5. 关系降温 | 二人继续合作，但交流变得程序化。 | 数据目录的价格字段被解密。 |

### 情感与专业控制

- 本章不靠误会拖戏，分歧来自真实职责与价值观。
- 专业点：数据所有权、授权、法律结论与事实描述。
- 章尾：地球不同危机与合作数据被按稀缺度明码标价。

## 第 070 章：价格表上的文明

### 章节任务

- 完整揭示非法文明决策数据交易。
- 将轻松爽文短暂切入严肃悬疑核心。
- 埋设监管体系买方名单。

### 场景拆分

| 场景 | 主要动作 | 结果 |
|---|---|---|
| 1. 价格目录 | 团队看到战争、灾害、合作、技术突破被分类定价。 | 地球长期被当作决策数据源。 |
| 2. 人为放大 | 唐可证明部分极端事件被节点轻微放大以提高数据价值。 | 非法行为造成真实伤害。 |
| 3. 收益链 | 芙娅追踪付款摘要，归零服务是主要结算方。 | 动机从维护便利转为长期利益。 |
| 4. 买方名单 | 部分买方字段指向政策预测机构，其中有监管关联代码。 | F-012 埋设。 |
| 5. 证据封存 | 陈默要求先保存原始目录，不立即公布。 | 贸然公开可能触发灭证。 |

### 基调要求

- 本章显性笑点最多一个，且只能来自价格表使用项目管理术语包装灾难，不能拿受害经历开玩笑。
- 专业点：数据商品、收益链、原始证据保全。
- 章尾：芙娅按程序提交举报，系统却把举报转交归零服务联合体自查。

## 第 071 章：一份无法正常提交的举报

### 章节任务

- 展示制度回路如何被利益方控制。
- 让芙娅第一次明确站在修复程序本身的位置，而非机械执行。
- 推动承包商正式回应。

### 场景拆分

| 场景 | 主要动作 | 结果 |
|---|---|---|
| 1. 正式举报 | 芙娅提交数据抽取、目标覆盖工具关联和灭证风险。 | 系统自动分配给合同责任方自查。 |
| 2. 利益冲突 | 责任方正是归零服务联合体。 | 举报对象成为调查入口管理者。 |
| 3. 改走紧急通道 | 芙娅尝试事故证据保全通道。 | 需要监管案件编号，而预审尚未立案。 |
| 4. 陈默的工单经验 | 陈默建议拆分成设备安全、数据异常和权限滥用三项可受理事实。 | 至少一项进入外部审计队列。 |
| 5. 调离压力 | 芙娅因“重复提交”收到正式警告。 | 她仍保留全部提交记录。 |

### 笑点与专业

- 笑点：举报系统要求被举报单位确认是否同意接受举报；宇宙流程也会把问题退回“请联系责任部门”。
- 专业点：利益冲突、受理条件、把大问题拆成可进入系统的事实项。
- 章尾：归零服务发来标准回复，要求地球证明自己对数据拥有权利。

## 第 072 章：承包商的标准回复

### 章节任务

- 让归零服务以制度与合同形式正式进入对抗。
- 泄露事故前生成的地球终止预案编号。
- 保持赛珞本人尚不直接出场。

### 场景拆分

| 场景 | 主要动作 | 结果 |
|---|---|---|
| 1. 合法备份说 | 对方声称节点是合规灾备设施，数据属于项目所有者。 | 回避节点主动制造异常的问题。 |
| 2. 权利证明 | 要求地球提交项目法人和数据授权文件。 | 地球在外部法律结构中没有对应主体。 |
| 3. 技术附件 | 陈默逐项核对附件，发现节点协议与所谓备份规范不符。 | 回复无法解释目标覆盖水印。 |
| 4. 误带编号 | 附件元数据含“E-731 清洁终止预案”。 | 文件创建时间早于试射事故。 |
| 5. 请求原件 | 团队正式要求预案原文和审批链。 | 对方随即停止回复。 |

### 笑点与专业

- 笑点：承包商要求地球出示数据所有权证明，陈默一时不知道该由谁给八十亿人盖章。
- 专业点：合同声称、技术事实、元数据时间戳。
- 章尾：多个节点同时进入高温自毁模式。

## 第 073 章：节点开始自毁

### 章节任务

- 启动后半卷行动高潮。
- 明确有人仍能远程控制节点。
- 让“救人还是保证据”成为现实决策，但拒绝残酷伪命题。

### 场景拆分

| 场景 | 主要动作 | 结果 |
|---|---|---|
| 1. 热失控 | 节点过载并向周边现实参数释放扰动。 | 城市基础设施风险快速升高。 |
| 2. 自动灭证 | 存储区开始清理数据，上传通道关闭。 | 明显是预设自毁程序。 |
| 3. 两种目标 | 完整保留数据需维持节点运行，立即切断才能降低人员风险。 | 团队出现争议。 |
| 4. 安全隔离 | 陈默决定先切断危险接口，接受部分数据损失。 | 多数节点停止，但证据只剩摘要。 |
| 5. 最大节点 | 汇聚中心拒绝停机，根裂纹随温度扩大。 | 必须在下一章完成最终取舍。 |

### 基调与专业

- 危机中不安排轻松笑点，只保留系统用“清洁关闭”称呼自毁的冷反差。
- 专业点：热失控、故障隔离、安全优先级。
- 章尾：最大节点的原始目录将在九十秒后被全部擦除。

## 第 074 章：先救人，再保存校验值

### 章节任务

- 完成节点自毁危机。
- 体现陈默不拿居民安全换“完美证据”。
- 用最小摘要和校验值保留可验证结论。

### 场景拆分

| 场景 | 主要动作 | 结果 |
|---|---|---|
| 1. 证据最小集 | 陈默、芙娅确定只保留目录时间戳、付款摘要、协议水印和终止预案索引。 | 不再追求复制全部交易数据。 |
| 2. 生成校验值 | 唐可和乌尔澄在断电前生成不可伪造摘要。 | 后续可证明原文件曾存在且未被替换。 |
| 3. 切断节点 | 坎提斯与现场队伍隔离物理支撑和热接口，陈默关闭参数连接。 | 根裂纹停止扩大。 |
| 4. 数据损失 | 大部分详细买方记录被清除。 | 调查受损，但城市安全恢复。 |
| 5. 时间证明 | 保留摘要显示终止预案早于事故创建。 | 事故不是终止计划的原因。 |

### 笑点与专业

- 危机结束后可有一个轻微笑点：系统评价“证据保全率不足”，陈默要求它先统计人员保全率。
- 专业点：哈希/校验值、最小证据集、不可逆取舍。
- 章尾：陈默提出结论——如果地球被删除，剩余证据也会永久消失。

## 第 075 章：删除是一种灭证方式

### 章节任务

- 完成第三卷核心动机揭示。
- 把地球存亡从“受损样本处置”转成“事故证据保全”。
- 触发银河监管提前预审和权限冻结。

### 场景拆分

| 场景 | 主要动作 | 结果 |
|---|---|---|
| 1. 时间链复盘 | 团队排列节点部署、数据交易、终止预案、试射和自毁事件。 | 终止计划明显早于事故。 |
| 2. 动机假设 | 陈默提出事故可能被利用来让地球合法进入删除流程。 | 从技术缺陷调查升级为有组织灭证调查。 |
| 3. 证据边界 | 芙娅强调目前能证明灭证受益和工具关联，尚不能直接指认个人。 | 防止过早定案。 |
| 4. 保全申请 | 团队以“关键证据与事故证人载体”为由申请暂停删除。 | 请求首次进入监管紧急队列。 |
| 5. 监管回应 | 银河监管局宣布提前预审，冻结大部分现实修改权限。 | 第四卷启动。 |

### 基调与专业

- 笑点只放在开场复盘：终止预案的流程状态写着“事故后待启动”，创建时间却在事故前。
- 专业点：时间因果链、动机假设与证据保全。
- 卷末钩子：闻铎尚未到场，权限冻结通知先一步抵达，地球多个补丁进入无法续期状态。
