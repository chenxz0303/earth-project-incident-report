# 《地球项目事故调查报告》项目说明书

## 项目目标

建立一套适合长期维护的纯静态小说发布系统，同时服务网页阅读、安卓“阅读/Legado”导入和长篇创作管理。目标篇幅约 50 万字，计划 5—7 卷、140—170 章，每章约 3000—4000 字。

当前阶段只建设网站、数据接口、书源、自动部署、测试和创作档案，不开始正式小说正文。

## 已确定信息

- 作品名：《地球项目事故调查报告》
- 作者显示名：陈默项目组
- 语言：简体中文
- 用途：个人阅读
- 发布方式：GitHub Pages 静态网站；Gitee 提供国内 Legado 数据镜像
- 数据方式：书籍信息、目录和章节均使用 JSON
- 测试内容：只保留一篇明确标注为“非正式正文”的测试章

## 技术结构

- `index.html`：作品首页、简介与分卷目录。
- `reader.html`：统一章节阅读页，通过固定章节参数加载 JSON。
- `data/book.json`：机器可读的书籍信息。
- `data/catalog.json`：嵌套分卷目录和供书源使用的扁平章节目录。
- `chapters/`：每章一个独立 JSON 文件。
- `legado/book-source.json`：安卓“阅读”书源。
- `scripts/check-project.mjs`：JSON、链接、目录和书源一致性检查。
- `.github/workflows/pages.yml`：自动测试并部署 GitHub Pages。

- 2026-07-11：建立公开 Gitee 国内数据镜像，书源、目录和章节数据改用 Gitee Raw 固定地址，减少国内网络访问 GitHub Pages 的依赖；GitHub 保持主仓库。

## 长期维护约定

正式章节完成后，必须在同一次改动中更新：

1. `data/catalog.json`；
2. `docs/章节摘要.md`；
3. `docs/时间线.md`；
4. `docs/人物状态.md`；
5. `docs/伏笔表.md`；
6. `docs/世界观规则.md` 中受到影响的关键设定。

所有正式创作开始前，先删除或替换非正式测试章，并保证测试脚本通过。
